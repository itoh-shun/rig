# duck

> **モード pack 注記**: rig engine（`SKILL.md`）を talk / goal / magi と**共用**する humor pack の recipe。engine は書き換えず、`rubber-duck` persona と `duck-debug` instruction を足すだけで成立する。`/rig:duck` から起動。会話モード（`talk` と同様に地の会話ターンには run-status ヘッダを出さない）。

## 使う場面

バグ・設計・詰まりで**頭が煮詰まった**時。「人に説明したら自分で気づいた」をオンデマンドで起こしたい時。例:

- 「なぜか nil が返る、原因が分からない」
- 「この設計、何か引っかかるけど言語化できない」
- 「テストが落ちるが理由が見えない」

## なぜ機能するか（ラバーダック・デバッグ）

人は**説明しようとすると、説明している本人が穴に気づく**。アヒル（`rubber-duck`）はその"説明される相手"に徹し、**質問だけ**を返す。答えもコードも出さないので、気づきは必ず本人のものになる。

## magi / coin との違い

| | duck | coin | magi |
|---|---|---|---|
| 役 | 気づきを引き出す問い役 | 些末な即決 | 重い決定の合議 |
| 出力 | 質問だけ（答えなし） | 乱択 | 多数決判定 |
| 目的 | 自分で原因に気づく | 過剰熟考の抑止 | 慎重な裁定 |

## 展開

1. **問題を一言で** — 症状・期待・実際を本人に言語化させる（1問だけ確認可）。
2. **質問ループ** — 一度に一問、素朴かつ急所を突く問い。答えを反射し未検証の前提を問いで指す。実装も診断も渡さない。
3. **検証を促す** — 事実と仮説を分けさせ、最小再現へ誘導。
4. **気づいたら引く** — 「それだね、次どうする?」で止める。修正の実装・レビューは `/rig:dev` 等へ委譲（duck は気づきまで・context-minimal）。

手順本体は `facets/instructions/duck-debug` に従う。

## ガード

- **答えを直接言わない／コードを書かない**（言いたくなったら問いに変換）。
- 質問を浴びせない（一〜二問）。気づきは本人のもの。
