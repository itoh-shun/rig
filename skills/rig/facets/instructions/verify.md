# instruction: verify

ビルド・lint・テストを実行して変更が壊れていないことを確認する。

## 手順

### ① コマンドの解決

実行するコマンドは以下の優先順で決定する：

1. **manifest にコマンドが定義されている場合** — manifest の `verify` または `check` セクションに従う
2. **manifest に定義がない場合（自動検出）** — プロジェクトルートを探索して以下の順に判定する：
   - `package.json` が存在する → `npm run build` / `npm run lint` / `npm test`（またはそれに相当するスクリプト）
   - `Makefile` が存在する → `make test`（またはそれに相当するターゲット）
   - その他ビルドツールが検出された場合は同様に推論する

### ② 実行

解決したコマンドを順に実行する。いずれかが失敗した場合は次ステップへ進まず、エラー内容を implement フェーズへフィードバックする。

### ③ ビジュアル確認（条件付き）

`--visual` フラグが付いている、または変更に UI 差分が含まれる場合は、verify 完了後に `visual-verify` ステップを続けて実行する。

### ④ 結果の引き継ぎ

全チェックが通ったら「verify 済み」として pr ステップへ進む。
