# instruction: call-script

開発資材 → 荷電（コールドコール）スクリプトの生成 **routing**。電話の構え（15秒で価値・反論武装・next action）は委譲先 persona（`sales/cold-caller`）が持つのでここには再掲しない（Native-first）。

**スコープ**: 開発資材（または先に作った営業1枚資料）から、電話営業で使える**荷電スクリプト**を生成する。オープニング15秒・ヒアリング・価値提示・反論処理・クロージングまで。

## 手順

1. **材料の収集** — 開発資材（`README*` / `CHANGELOG*` / `plugin.json` / `docs/`）と、あれば既存の営業1枚資料を集める（`--from <path>` で対象指定可）。長文は親 context に引き込まない（context-minimal）。
2. **固有知識の注入** — `facets/knowledge/sales-domain/`（ICP・刺さるトリガー・競合の勝ち筋）があれば、フックと反論処理の材料に渡す。無ければ汎用で書き、ICP/トリガーは仮説＋`[要記入]`。
3. **生成の dispatch** — `sales/cold-caller` を合成して subagent に渡す。**話し言葉・15秒オープニング・課題喚起の質問・実プロダクトの強みで返す反論処理・next action でクロージング**のスクリプトを書かせる。相手依存箇所は `[会社名]`/`[業界]`/`[具体課題]` でプレースホルダ化させる。
4. **構造化** — 出力は `output-contracts/sales-collateral`（B. 荷電スクリプト）に従わせる。

## ガード

- **捏造の強みで反論を返さない**（実在機能の範囲で切り返す）。
- その場で売ろうとせず **next action（アポ取得等）** をゴールにする。
- 話し言葉で（書き言葉・専門用語の連発を避ける）。相手依存はプレースホルダで埋めた風にしない。
