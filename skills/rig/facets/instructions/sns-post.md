# instruction: sns-post

X のポストを**半自動**で起案する手順。アカウントの声は運用者の声 persona（`/rig:persona` 製・`--persona`/`default_personas` で投入）が、判断は `personas/sns-post-reviewer` が、事実は `knowledge/sns-x-conventions` ＋ `knowledge/ai-writing-smells` が持つ（Native-first）。主たる用途は歌ってみた等クリエイターの宣伝運用。

**標準スコープ**: 文脈/トリガー取り込み → 切り口 → ドラフト（声 persona）→ AI 臭除去 → レビュー＆分類 → 定型は承認キュー / 要判断は停止して承認。

## 入力（トリガー）
次のいずれかを文脈として受ける：
- **告知系**：新曲の公開予定/公開直後（曲名・日時・聴きどころ・サムネ・リンク先）。
- **裏側系**：制作中の進捗・選曲理由・録り直し等。
- **交流/日常系**：他クリエイターへの言及、音楽観、近況。
- **計測系**：直近投稿の反応（伸びた/伸びない）— 次の切り口に反映。

## 手順

1. **文脈収集** — トリガーと必要事実（確定した日時・曲名・出典）を集める。**未確定の事実は捏造しない**（要判断へ）。
2. **切り口** — 型（`sns-x-conventions` のテンプレ）から1〜数案。宣伝:日常のバランス（≒2:8）を意識。
3. **ドラフト** — **声 persona** で起案（agent/persona 合成）。1行目で掴む・リンクは末尾/リプへ・ハッシュタグ1〜2個。
4. **AI 臭除去** — `de-ai-smell`（`knowledge/ai-writing-smells`）を効かせ、誇張マーケ語・水増し・鋳型を落とす。**声の人間味は均さない**。
5. **レビュー＆分類** — `sns-post-reviewer` で掴み/ブランド/AI臭/リスク/導線を判定し、**定型 / 要判断**に分類（agent 優先、無ければ `facets/personas/sns-post-reviewer` 合成）。
6. **収束とルーティング** — `patterns/acceptance-gate` で「掴み有・ブランド適合・AI臭なし・低リスク」へ収束。
   - **定型**：gate 通過 → **承認キューへ**（投稿時間の提案つき）。
   - **要判断**：**停止して運用者へエスカレーション**（理由＝権利/比較/事実主張/新規性等を明示）。
7. **出力** — `output-contracts/review-verdict` ＋ 投稿本文・投稿時間提案・分類・リスクメモ。**実投稿は別アダプタ**（手動キュー / X API。本手順は承認可能な状態まで）。

## 注意
- 対外・本人名義。**迷ったら要判断**に倒す。
- 事実（公開日時・出典・数字）は裏取りできないものを断定させない。
- 連投・宣伝の繰り返しを避ける（同一告知は表現を変え間隔を空ける）。
