# instruction: release-movie

リリースムービー（リリーストレーラー）生成の **routing**。演出の作法（ハイプだが嘘なし・構成・テンポ）は委譲先 persona（`release-director`）が持つのでここには再掲しない（Native-first）。

**スコープ**: CHANGELOG / リリースノート / git ログから、ソフトウェアリリースの**短いトレーラー**を作る。納品は 2 つ — ①制作台本（絵コンテ）と ②再生できるアニメ HTML トレーラー。

> ⚠️ harness は**実動画をレンダリングしない**。納品は「制作台本（実動画編集に渡せる）」＋「ブラウザで再生できる HTML トレーラー（実物のアニメーション）」。実 mp4 が要る場合は台本を After Effects / CapCut 等に渡す前提。

## 手順

1. **素材の収集** — 対象リリースを確定する。既定は **CHANGELOG の最新エントリ**（`## [x.y.z]`）。引数でバージョン/タグ指定可。補助に `plugin.json` の description・直近の git ログ（前タグ以降）・README の要点。長文は親 context に引き込まない（context-minimal）。
2. **目玉の決定** — 今回のリリースの**一番の価値**を 1 つ選ぶ（クライマックスに置く）。
3. **生成の dispatch** — `release-director` を合成して subagent に渡し、下記2点を作らせる。
4. **接続（任意）** — 台本のコピーは `/rig:dev --recipe de-ai-smell` でさらに磨ける。

## 納品物 ①：制作台本（正準フォーマット）

```
## リリースムービー台本: <product> <version>

ログライン: <30字以内の惹句>
尺: 合計 <N> 秒 ／ 音楽: <テンポ・雰囲気>

### シーン表
| # | 尺 | 種別 | 映像/演出 | テロップ(画面の文字) | VO(ナレーション) | BGM/SE |
|---|---|---|---|---|---|---|
| 1 | 3s | cold-open | … | … | … | … |
| … |

### CTA（最終カード）
<バージョン・入手方法・一言>

### ソース対応表（誇張防止）
- シーン<#> → CHANGELOG「<項目>」  …（各目玉カットが実機能に紐づくことを示す）
```

## 納品物 ②：アニメ HTML トレーラー

`web/release-trailer.html` の **SCENES 配列**を、台本に合わせて埋める（または同梱テンプレを複製して書き換える）。SCENES の各要素は：

```js
{ type: "title|feature|stat|list|cta", dur: <秒>,
  title: "...", subtitle: "...", items: ["...","..."], accent: "#hex" }
```

- `type` ごとに演出（title=タイトル、feature=機能リビール、stat=数字、list=箇条、cta=締め）。
- `dur` は表示秒（台本の尺と一致させる）。ブラウザで開くと SCENES が順に自動再生される（再生/停止・前後・リプレイ・任意 BGM つき）。
- ロジック（プレイヤー）は既存。**埋めるのは SCENES データだけ**（HTML を一から書かない）。

## ガード

- **各ビートを実機能に紐づける**（ソース対応表必須）。出所の無い派手な主張・捏造機能を作らない。
- **空ワード禁止**（「革命的」「次世代」等）。具体・数字・before→after で熱を作る。
- 目玉は 1 つに絞る（全部を主役にしない）。テロップは 1 行・数語。
