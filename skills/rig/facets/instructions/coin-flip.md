# instruction: coin-flip

コイン投げ即決の **routing**。判断の構え（トリアージ規則）は委譲先 persona（`coin-flipper`）が持つのでここには再掲しない（Native-first）。

**スコープ**: 可逆で些末な 50/50（または N 択）を、熟考させずに即決する。bikeshedding の抑止。`magi`（重い・不可逆な決定の合議）の対極。

## 手順

1. **議題の確定** — 何を決めるか（2択 or 選択肢リスト）を1つに確定する。曖昧なら**1問だけ**確認。
2. **トリアージ（`coin-flipper` の構え）** — 可逆性・被害半径・どちらでも実害小、の3点を判定する。
   - **重い/不可逆と判明したら投げない** → `/rig:magi <議題>` へ回す（「これはコイン案件じゃない」と明示）。コインで重大決定を下さない（これが最大のガード）。
3. **投げる** — 些末で可逆と確認できたら、公平に乱択して1つに確定する。正準出力（下記）で提示。
4. **背中を押す** — 「確定・可逆だからまず動こう」と添えて、再議論のループに戻さない。

## 正準出力

```
## rig coin

議題: <決めること>
トリアージ: 可逆 ✓ / 被害半径 小 ✓ / どちらでも実害小 ✓

      ┌───────┐
      │  <結果>  │   ← 表 / 裏（または選んだ選択肢名）
      └───────┘

確定: <結果>。可逆だからまず動こう — ダメなら戻せばいい。
```

トリアージで弾かれた場合は投げず、次の形で回す：

```
## rig coin → magi 案件

この決定は可逆/些末ではありません（理由: <不可逆 / 被害半径大 / 実害大>）。
コインで決めるべきではない。→ `/rig:magi <議題>` で3賢者に諮ってください。
```
