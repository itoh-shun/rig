# instruction: verify-commands

**documentation 専用。** ドキュメント中のコマンド例・コードブロックが実際に動くことを確認する。「READMEのコマンド例が動かなくなる変更」（`recipes/drill` の種カタログにも載る典型的なドキュメント虚偽化）を機械的に防ぐ。

## 手順

### ① コマンド例の抽出

`docs-draft` が書いた（または既存の）Markdown からコードブロック（` ``` ` で囲まれたシェルコマンド・スクリプト片）を抽出する。

### ② 実行確認

抽出した各コマンドを、**隔離 worktree 内で**（本体を汚さない）実行できる範囲で実行し、以下を確認する：

- コマンドが exit 0 で終わるか
- 出力例が示されている場合、実際の出力と大きく乖離していないか
- 破壊的なコマンド（`rm -rf` 等）・外部への副作用があるコマンド（デプロイ・課金操作等）は実行せず、**構文とオプションの妥当性のみ静的に確認**する（実行不可の旨と確認方法を note に残す）

### ③ 不一致の扱い

コマンドが失敗した、または出力例が古い場合は `docs-draft` へ差し戻して修正する。実行確認できなかった項目（外部サービス依存等）は `acceptance-check` step の `tests_pass_or_reasonable_explanation` 判定に「実行未確認・理由: …」として引き継ぐ（`warn` 扱い）。
