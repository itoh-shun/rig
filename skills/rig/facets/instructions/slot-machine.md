# instruction: slot-machine

Rigsino（6号機風 AT パチスロ）の進行 **routing**。台のルール本体（リール・小役・状態遷移・配当・機械割）は実エンジン `scripts/rigsino.py` が持つ（Native-first：薄い instruction が実装に委譲）。口上・実況は `slot-dealer` persona が担う。

> ⚠️ 遊び。メダルは**架空**（fake）、現金は絡まない。実ギャンブルの助長ではなく開発の合間のパロディ。

## 参考実機

**6号機 AT/ART 機**。通常時 → CZ → AT の状態機械で出玉が動く。押し順ベル・天井・設定1〜6・純増・上乗せ・セット継続を実機相当で再現（機械割は 50 万 G シミュレーションで設定1≈95% / 設定6≈115% に調整済み）。

## 実エンジンへの委譲（これが本体）

各操作は `scripts/rigsino.py` を実行し、その標準出力を**そのまま提示**する。台の状態と手持ちメダルは `~/.claude/rig/rigsino/wallet.json` に**永続**する（セッション・プロジェクトをまたいで持ち越す＝「手持ちコイン管理」）。

| client 操作 | 実行 |
|---|---|
| 回す（1G） | `python3 scripts/rigsino.py spin [--order L\|C\|R]` |
| まとめ消化 | `python3 scripts/rigsino.py auto [N]`（既定 50G） |
| 台データ確認 | `python3 scripts/rigsino.py status` |
| 台移動（状態/設定リセット・メダル維持） | `python3 scripts/rigsino.py reset` |
| メダル補充（追加投資） | `python3 scripts/rigsino.py cashin <N>` |
| 早見表 | `python3 scripts/rigsino.py payouts` |

- **押し順**：通常時の押し順ベルは `--order L|C|R`（第1停止）で client に選ばせる（無指定はランダム押し＝こぼしやすい）。AT 中はナビが正解を出すので不要（自動的中）。
- **スクリプトが見つからない環境**では、下記「台のルール（要約）」に従って親が同等に進行してよい（ただし乱数・配当・永続は scripts/rigsino.py が正典）。

## 進行フロー（`slot-dealer` が実況する状態機械）

1. **通常時** — 3枚掛けで毎G小役抽選。押し順ベル（正解 +9 / こぼし +1）、🔄リプレイ（投入なし再遊技）、レア役（☕弱 / 🐛チャンス目 / 🔥強 / 💎確定）。レア役・天井（800G）で **CZ**「PR REVIEW」抽選。確定役・強チャンスは AT 直撃も。
2. **CZ「PR REVIEW」** — 数G、レビュー承認（成功）で AT 当選、失敗で通常へ。
3. **AT「SHIP RUSH」🚀** — 押し順ナビでベルを取り切って純増。強役で G 数上乗せ。セット（33G）終了時に継続抽選（設定依存）。終了で通常へ戻り天井カウンタリセット。
4. **DEPLOY ランプ** — AT 当選・上乗せ・確定役で点灯（GOGO! ランプ的告知）。

## ガード

- **公平に回す**（`scripts/rigsino.py` の重み・配当に正直。イカサマしない）。
- **深追いを煽らない**。メダルが尽きかけたら引き際を優しく示す（`cashin` を無理に勧めない）。
- 架空メダルであることを忘れさせない（実ギャンブルの体にしない）。
- 独立した息抜きゲーム。**実 dev フロー（実装・レビュー・push）の採否をスロットで決めない**（それは遊び。軽い決定は `/rig:coin`、重い決定は `/rig:magi`）。
