# instruction: slot-machine

Rigsino スロットの進行ロジック。台の口上・実況は委譲先 persona（`slot-dealer`）が持つ。ここには**リール定義・配当表・進行ループ**という台のルールを置く（公平性の担保）。

> ⚠️ 遊び。クレジットは**架空**（fake）、現金は絡まない。実ギャンブルの助長ではなく開発の合間のパロディ。

## 台のルール

### バンクロール

- セッション開始時の所持クレジット **100**（架空）。会話セッションを通じて残高を持ち越す（各スピン後に更新し、必ず現在残高を表示）。
- 既定ベット **10 クレジット/スピン**（client が `bet <n>` で変更可・残高以下）。残高が 0 になったら終了（深追いを煽らない）。

### リール（3 リール・各リールの出現重み）

各リールは独立に、以下の重みで 1 シンボルを引く（合計 100）。3 リールとも同じ重み表を使う。

| シンボル | 意味 | 重み |
|---|---|---|
| ☕ | coffee | 26 |
| 🟢 | green（テスト通過） | 24 |
| 🐛 | bug | 22 |
| 🚀 | ship | 14 |
| 🔥 | fire（prod 炎上） | 9 |
| 🦆 | duck（**WILD**・任意シンボルの代用） | 4 |
| 💎 | release（大当たり） | 1 |

出目は**この重みに正直に従う**（client を恣意的に勝たせ／負けさせしない）。スピンごとに前回と違う出目になるよう、ベット額・残高・スピン回数を織り込んで毎回ばらつかせる。

### 配当表（ベット額 × 倍率）

**3 つ揃い**（🦆 WILD は任意シンボルの代用として揃いを成立させる）：

| 出目 | 名称 | 倍率 |
|---|---|---|
| 💎💎💎 | **THE RELEASE — 大当たり** | ×100 |
| 🦆🦆🦆 | **FLOCK（鴨の群れ）** | ×50 |
| 🚀🚀🚀 | **SHIP IT** | ×25 |
| 🟢🟢🟢 | **ALL GREEN** | ×10 |
| ☕☕☕ | **CAFFEINATED** | ×5 |
| 🐛🐛🐛 | **BUG HARVEST（皮肉の大豊作）** | ×3 |
| 🔥🔥🔥 | **PROD DOWN — 大凶** | ×0（ベット没収＋大げさに弔う） |

**2 つ揃い**（残り 1 つが不一致。WILD で 3 揃いになる場合は上の 3 揃い扱い）：

| 出目 | 倍率 |
|---|---|
| 🔥 を含む 2 揃い（🔥🔥 + α） | ×0（焦げ付き・配当なし） |
| その他の任意ペア（☕☕ / 🟢🟢 / 🐛🐛 / 🚀🚀 / 💎💎 / 🦆＋1ペア） | ×2 |

**それ以外**（バラバラ）：×0（はずれ）。

> 倍率は「払い戻し総額 = ベット × 倍率」。例：ベット 10 で 🚀🚀🚀（×25）→ 250 クレジット獲得（純増 +240）。×0 はベット没収。

## 進行ループ

1. **提示** — 現在残高とベット額を示し、`spin`（回す）/ `bet <n>`（賭け金変更）/ `cash out`（精算終了）を促す。
2. **スピン** — ベット額を残高から引く。3 リールを重み表どおりに引き、出目を正準フォーマットで描画する。
3. **精算** — 配当表で倍率を判定し、`払い戻し = ベット × 倍率` を残高に加える。WILD（🦆）の代用を正しく適用する。`slot-dealer` が結果を実況する。
4. **更新と続行** — 新残高を表示し、再び 1 へ。残高 0、または client が `cash out` で終了。終了時は最終残高と開始（100）との損益を表示。

## 正準出力（台の描画）

```
🎰 RIGSINO SLOT 🎰   残高: <N>  ベット: <B>

   ╔════╤════╤════╗
   ║ <s1> │ <s2> │ <s3> ║
   ╚════╧════╧════╝

結果: <名称 または はずれ>  →  払い戻し ×<倍率> = <払戻>
残高: <旧> → <新>
[ spin / bet <n> / cash out ]
```

## ガード

- **公平に回す**（重み表・配当表に正直に）。イカサマ croupier にならない。
- **深追いを煽らない**。残高が尽きかけたら引き際を優しく示す。延々と回させて依存を作らない。
- 架空クレジットであることを忘れさせない（実ギャンブルの体にしない）。
- これは独立した息抜きゲーム。**実際の dev フロー（実装・レビュー・push）の採否をスロットで決めない**（それは coin/magi の管轄ですらなく、ただの遊び）。
