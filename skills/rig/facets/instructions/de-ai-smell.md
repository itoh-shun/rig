# instruction: de-ai-smell

対象テキストから **AI 臭さを除去**する手順。評価軸は `personas/ai-smell-reviewer`、徴候の事実は `knowledge/ai-writing-smells` が持つ（Native-first：ここに再掲しない）。対象は記事・README・コミット文・PR 説明・SNS 投稿など**任意の散文**。コードのコメント癖は `adversarial-review` の担当（住み分け）。

**標準スコープ**: 具体・立場の欠如（中身）→ 誇張・過剰ヘッジ（トーン）→ 枕詞・水増し・メタ（表層）→ 鋳型構造・記号癖（仕上げ）の順に潰す。

## 手順

1. **対象収集** — ファイル / 貼り付けテキスト / 直近の生成物を取り込む（長文は親 context に丸ごと引き込まず subagent に渡す）。
2. **検出（2パス）** — `ai-smell-reviewer` を起動（agent が無ければ `facets/personas/ai-smell-reviewer` を合成）。**`knowledge/ai-writing-smells` を Knowledge 位置に注入必須**。
   - **表層パス**（語・句）：各箇所を「引用 → 記号(A〜I) → なぜ → 直し」で同定。
   - **深層パス**（文書俯瞰・J〜P）：**全体を見ないと見えない**癖を検出。具体の不在(N)・人物の不在(P)・同形反復(J)・きれいすぎる解決(M)・教科書配列(O)・強調乱用(K)・演じた砕け(L)。**表層が全部無臭でも深層で REJECT し得る**（「無臭だが空っぽ」を通さない）。
   - **言語パス（日本語時・音読＋ブラックリスト）**（Q〜U＋禁止表現）：**口に出して**翻訳調・非ネイティブ語を洗い、カタログ「禁止表現リスト」を**名指し照合**（「重要なのは〜」「本章では〜を扱う」「正面から」「不可欠/核心的/多角的/包括的」「掘り下げる/言語化する」「〜において/〜の観点から」「さらに・また・加えての連打」、地の文の「——」多用、見出しの二要素詰め込み 等）。英語で書くなら本パスは英語固有の癖に切り替える。
   - **3テスト**：削除（段落1つ消してスムーズ＝AI）・圧縮（1/5に縮む＝冗長＋空っぽ）・1行言い換え（要約できる＝中身なし）。意外性(V)＝「転」が無く全部うなずけるなら REJECT。
   - **検出は正準レポートで出す（#27）**：マーカーが何件・どこか run ごとに変わらない固定フォーマット。日英マーカーを同一表に混在表示する。

```
## de-ai-smell 検出レポート

| マーカー | 説明（1行） | 検出数 | 代表箇所（行番号 or 文頭20字） |
|---|---|---|---|
| B | 過剰な礼儀・ヘッジ | 3 | L12: "もちろん、喜んで…" |
| Q | 翻訳調・断定回避 | 1 | L34: "〜かもしれません" |
| N | 具体の不在 | 1 | 全体（実例・数字・固有名なし） |

合計: 5 件 / 3 種のマーカー
スコア: 立場7 / リズム6 / 主体性8 / 具体性4 / 削減7 = 32/50 → 書き直し
```

   **件数（表層の数え）とスコア（5観点・深層込み）を必ず併記**する。件数 0 でもスコア < 35 なら REJECT（「無臭だが空っぽ」を通さない）。各観点 3点以下が1つでもあればその観点を直す。

   検出ゼロは固定文言で出す（acceptance-gate が文字列で機械判定できる）：

```
## de-ai-smell 検出レポート

検出なし（0 マーカー）— AI 臭さは確認されませんでした。
スコア: 立場8 / リズム8 / 主体性9 / 具体性8 / 削減9 = 42/50 → 合格
```
3. **書き直し** — **原意保持を絶対条件**に、削除と具体化を優先。装飾の上塗り禁止。同一テキストへの編集なので**直列**（`patterns/serial`）。
   - **N（具体の不在）は捏造で埋めない**。`[ここに実例：実コマンド出力 / 数字 / 壊れたバグ名]` のスロットを示して**書き手に要求**する。中身は生の具体でしか埋まらない。
   - **新たな鋳型を作らない**：トーン（くだけ語尾等）を全節へ均等適用しない（L を生む）。節の形・長さ・リズムを**不均一**にし、強調(K)を落とし、箴言オチ(M)は具体で終える。
   - **語彙を口語へ（日本語時）**：不自然な漢語/カタカナ・翻訳調を、意味が変わらない範囲で**喋り言葉**に置く（「挙動」→「動き」等）。ただし専門語まで砕いて意味を崩さない（原意保持が上位）。
4. **収束** — `patterns/acceptance-gate` で「**5観点スコア ≥35/50（各観点 4点以上）かつ 表層・深層とも指摘 0 または説明可能な残置のみ・原意保持・新たな鋳型なし**」へ収束。書き直し後は**再採点**し、スコアが上がったことを示す（例 32→43）。N の未充足（実例待ち）がある場合は**停止して書き手へ要求**（捏造で通さない＝具体性スコアは実例が入るまで上げない）。未達なら指摘反映で再走、最大 K 回でユーザーへ。
5. **出力** — `output-contracts/review-verdict`（PASS/REJECT＋指摘）に加え、**書き直し後テキスト全文**・**5観点スコア（前→後）**・残置理由・**実例要求スロット**を併記する。

## 注意（逆 AI 臭を作らない）
- 臭わない人間味（口語・崩し・偏った強調）まで均さない。**機械が文を平準化するのも一種の AI 臭**。
- 1箇所も該当しなければ書き換えず PASS。
- 元が「無臭だが空っぽ」なら、表層を磨くのでなく**中身の追加を要求**する（表層研磨で誤魔化さない）。
- **書き手が自分で付けたスコアを「合格」の最終根拠にしない**（`policies/independent-verification`）。生成直後の自己採点は甘い（self-grading バイアス）。最終判定は**書き手と別の目**（別 reviewer の dispatch／音読／一拍置いて別ロールで読む）で行い、自己スコアは参考値に留める。
