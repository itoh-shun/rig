# instruction: de-ai-smell

対象テキストから **AI 臭さを除去**する手順。評価軸は `personas/ai-smell-reviewer`、徴候の事実は `knowledge/ai-writing-smells` が持つ（Native-first：ここに再掲しない）。対象は記事・README・コミット文・PR 説明・SNS 投稿など**任意の散文**。コードのコメント癖は `adversarial-review` の担当（住み分け）。

**標準スコープ**: 具体・立場の欠如（中身）→ 誇張・過剰ヘッジ（トーン）→ 枕詞・水増し・メタ（表層）→ 鋳型構造・記号癖（仕上げ）の順に潰す。

## 手順

1. **対象収集** — ファイル / 貼り付けテキスト / 直近の生成物を取り込む（長文は親 context に丸ごと引き込まず subagent に渡す）。
2. **検出** — `ai-smell-reviewer` を起動（agent が無ければ `facets/personas/ai-smell-reviewer` を合成）。**`knowledge/ai-writing-smells` を Knowledge 位置に注入必須**——これがこの手順の主目的。各箇所を「引用 → カタログ記号(A〜I) → なぜ → 直し方」で同定する。
3. **書き直し** — **原意保持を絶対条件**に、削除と具体化を優先して書き直す。装飾の上塗り禁止。中身（G 具体 / E 立場）が薄い箇所は捏造で埋めず「具体/立場が必要」と要求として残す。書き直しは同一テキストへの編集なので**直列**（`patterns/serial`。並列書き換えはしない）。
4. **収束** — `patterns/acceptance-gate` で「**指摘 0 または説明可能な残置のみ・原意保持・不自然な定型なし**」へ収束。未達なら指摘反映で再走、最大 K 回でユーザーへ。
5. **出力** — `output-contracts/review-verdict`（PASS/REJECT＋指摘）に加え、**書き直し後テキスト全文**と、残置した臭い（直すと原意が変わるもの）の理由を併記する。

## 注意（逆 AI 臭を作らない）
- 臭わない人間味（口語・崩し・偏った強調）まで均さない。**機械が文を平準化するのも一種の AI 臭**。
- 1箇所も該当しなければ書き換えず PASS。
- 元が「無臭だが空っぽ」なら、表層を磨くのでなく**中身の追加を要求**する（表層研磨で誤魔化さない）。
