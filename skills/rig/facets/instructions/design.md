# instruction: design

設計ドキュメントを作成し、要件をピン留めする。`--design` フラグまたは recipe に design フェーズが含まれる場合のみアクティブになる。

## 手順

### ① 発動条件の確認

`--design` フラグが付いている、または recipe が design フェーズを含む場合にのみ本ステップを実行する。条件を満たさない場合はスキップして implement へ進む。

### ② 設計ドキュメントの作成

intake で確定したスコープを基に設計ドキュメントを作成する。最低限含める項目：

- 目的・背景
- 設計方針（インターフェース、データモデル、依存関係）
- 完了条件（AC）
- 除外事項

### ③ 要件の深掘り（`grill-with-docs` / `grill-me`）

設計ドキュメントを素材として `grill-with-docs` スキルを起動し、不明点・曖昧点を洗い出す。ドキュメントが存在しない場合は `grill-me` スキルでユーザーに直接問い直す。

### ④ 要件のピン留め

`grill-with-docs` / `grill-me` の出力を設計ドキュメントへ反映し、implement フェーズへ引き継ぐ最終 AC を確定する。
