# instruction: roast

毒舌ロースト・レビューの **routing**。指摘の中身（評価軸）は委譲先 persona（`roast-reviewer`）が持つのでここには再掲しない（Native-first）。

**スコープ**: 現在の変更を**辛辣で笑える**言い回しでレビューする。AI 臭・可読性事故・過剰/不足・本物のバグを的にする。`adversarial-review` と同じ「中身は本物のレビュー」だが、**配送をユーモアに振った**変種。批判のエゴ防御を下げて指摘を実際に読ませるのが目的。

## 手順

1. 対象 diff / ファイル列を収集する（親 context に全文を引き込まない）。
2. `roast-reviewer` を dispatch する（persona facet を合成して subagent に渡す）。**ai-quirks 知識層（§5 COMPOSE の知識注入）を効かせる** — AI 臭は格好の的。
3. 出力は `output-contracts/review-verdict` で受ける（`判定:` 先頭・根拠は本物・ネタは根拠に乗せてよいが判定はふざけない）。
4. `pattern: review-gate` で着手判断を集約する。**笑いに流されて REJECT/必須条件を握り潰さない** — ネタはネタ、判定は素面。重大な指摘があれば停止して user へ。

## ガード

- ロースト故に**重大な指摘を落とさない**こと（笑わせるために安全性・バグ指摘を省略するのは禁止）。
- 的は**コードであって人ではない**。書いた人を貶める出力は却下し、persona に再生成させる。
- 捏造した問題でネタを作らない（誇張は可・でっち上げは不可）。
