# output-contract: magi-verdict

## output-contract: magi-verdict

MAGI の各号機（Melchior-1 / Balthasar-2 / Casper-3）が共通で遵守する票の出力構造。`magi-consensus` ゲートが**判定を機械抽出**して多数決を集計するため、判定を先頭行に置き、前置き・挨拶・補足説明は禁止する。

### 形式

```
判定: <可決|否決|条件付可決|評価不能>
号機: <MELCHIOR-1|BALTHASAR-2|CASPER-3>（<科学者|母|女>）

核心: <1行。なぜその判定に至ったか>

評価:
1. （観点1）
2. （観点2）
3. （観点3）

条件:
- （条件付可決のときのみ。可決へ転じるための必須条件。可決/否決では条件ブロックごと省略）
```

`評価不能`（データ不足で判定を保留する場合）は形式が異なる：

```
判定: 評価不能
号機: <MELCHIOR-1|BALTHASAR-2|CASPER-3>（<科学者|母|女>）

不足情報: <1行〜数行。判断に必要だが得られていない情報・前提>
```

`核心:`/`評価:`/`条件:` は `評価不能` では省略する（判断していないため書けない。捏造しない）。

### ルール

- **判定を最初の行に必ず出力する**（`判定:` で始まる行）。判定語は `可決` / `否決` / `条件付可決` / `評価不能` のいずれか。
- **号機行**で自分がどの号機（どの人格）かを明示する（集計時に誰の票かを機械抽出するため）。
- **核心**は1行。判定の最大の理由を一文で言い切る（MAGI コンソールの要約に使われる）。`評価不能` では省略し、代わりに `不足情報:` を書く。
- **評価は必ず3点**（`評価不能` を除く）。各号機は自分の評価軸（Melchior=正しさ / Balthasar=守り / Casper=価値）に閉じて書く。他号機の領分（安全・価値・正しさ）に踏み込まない＝3票の独立性を保つ。
- **条件**は `条件付可決` のときのみ。可決に転じるための必須条件を箇条書きにする。`可決`/`否決`/`評価不能` では `条件:` ブロックごと省略する。
- **評価不能**は、判断に必要な情報・前提が欠けていて3値のいずれも選べない場合に使う（当て推量で `可決`/`否決`/`条件付可決` のいずれかを選ばない＝票を捏造しない）。`不足情報:` に何が足りないかを具体的に書く（`magi-consensus` が審議継続の再質問に使う）。
- 自分の票だけを出す。他号機の票を予想・代弁しない（合議は `magi-consensus` が集計する）。
- **全体 150–300字**（日本語換算。`評価不能` は `不足情報:` のみのため短くてよい）。冗長な前置き・感想・締め挨拶は禁止。冷たく、簡潔に。
