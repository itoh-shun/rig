# output-contract: magi-verdict

## output-contract: magi-verdict

MAGI の各号機（Melchior-1 / Balthasar-2 / Casper-3）が共通で遵守する票の出力構造。`magi-consensus` ゲートが**判定を機械抽出**して多数決を集計するため、判定を先頭行に置き、前置き・挨拶・補足説明は禁止する。

### 形式

```
判定: <可決|否決|条件付可決>
号機: <MELCHIOR-1|BALTHASAR-2|CASPER-3>（<科学者|母|女>）

核心: <1行。なぜその判定に至ったか>

評価:
1. （観点1）
2. （観点2）
3. （観点3）

条件:
- （条件付可決のときのみ。可決へ転じるための必須条件。可決/否決では条件ブロックごと省略）
```

### ルール

- **判定を最初の行に必ず出力する**（`判定:` で始まる行）。判定語は `可決` / `否決` / `条件付可決` のいずれか。
- **号機行**で自分がどの号機（どの人格）かを明示する（集計時に誰の票かを機械抽出するため）。
- **核心**は1行。判定の最大の理由を一文で言い切る（MAGI コンソールの要約に使われる）。
- **評価は必ず3点**。各号機は自分の評価軸（Melchior=正しさ / Balthasar=守り / Casper=価値）に閉じて書く。他号機の領分（安全・価値・正しさ）に踏み込まない＝3票の独立性を保つ。
- **条件**は `条件付可決` のときのみ。可決に転じるための必須条件を箇条書きにする。`可決`/`否決` では `条件:` ブロックごと省略する。
- 自分の票だけを出す。他号機の票を予想・代弁しない（合議は `magi-consensus` が集計する）。
- **全体 150–300字**（日本語換算）。冗長な前置き・感想・締め挨拶は禁止。冷たく、簡潔に。
