## output-contract: sales-collateral

開発資材から生成する営業資材の出力構造。2 種類（営業1枚資料 / 荷電スクリプト）の正準フォーマットを定義する。前置き・挨拶は禁止。**実在機能のみ・誇張禁止・不明は `[要記入: …]` プレースホルダ**を全体ルールとする。

---

### A. 営業1枚資料（`sales-material`）

```
# 営業資料: <プロダクト名>

## ヘッドライン
<一言の提供価値。誰の何を、どう変えるか。空ワード禁止>

## こんな課題ありませんか（ターゲットの痛み・3点）
- <痛み1> / - <痛み2> / - <痛み3>

## <プロダクト名> が解決します（機能→ベネフィット）
| 機能（実在） | だから何が嬉しいか（ベネフィット） | 出所 |
|---|---|---|
| <機能> | <消える痛み・得られる結果> | <README/CHANGELOG/コードの該当> |
（3〜5行）

## 差別化（競合・現状維持に対する勝ち筋）
- <他社/内製/現状維持 に対して、なぜこれか>

## 導入効果（before → after）
- <before の状態> → <after の状態>（定量できれば数字・無ければ定性。捏造しない）

## 想定顧客（ICP）
- 業種・規模: <sales-domain 知識があれば反映・無ければ [要記入]>
- 刺さるトリガー（なぜ今）: <…>

## 料金・導入
- プラン枠: [要記入: 価格レンジ]  ／  導入リードタイム: [要記入]

## 次の一歩（CTA）
- <デモ / トライアル / 資料DL 等、相手が踏める次アクション>
```

### B. 荷電スクリプト（`call-script`）

```
# 荷電スクリプト: <プロダクト名>

## 事前メモ
- 誰に: [業界][部署][役職]  ／  なぜ今: <トリガー仮説>

## 1. オープニング（〜15秒）
「<名乗り>。<電話理由>。<フック＝相手が"お、なに?"となる一言>」

## 2. 受付突破（ゲートキーパー）
「<担当者につないでもらう一言>」

## 3. ヒアリング（2〜3問・課題喚起）
- Q1: <現状を言わせる質問>
- Q2: <痛みを言わせる質問>

## 4. 価値提示（相手の課題に紐づけて一言）
「<課題> なら、<実在機能> で <結果>。」

## 5. 反論処理（よくある反論 → 切り返し）
| 反論 | 切り返し（実プロダクトの強みで） |
|---|---|
| 「今は不要」 | <…> |
| 「他社を使っている」 | <…> |
| 「予算がない / 忙しい」 | <…> |
（3〜5個）

## 6. クロージング（next action）
「<15分デモ / 資料送付＋再連絡日 等、具体的な次の約束>」

## 7. 終話
「<礼＋次回の確認>」
```

### 共通ルール

- **実在機能のみ**（README/CHANGELOG/コードに根拠）。捏造機能・盛った実績を書かない。
- **機能→ベネフィットに翻訳**（買い手は結果を買う）。技術仕様の羅列にしない。
- **AI 臭・空ワード禁止**（「革新的」「シームレス」「次世代」等）。具体・数字・固有名で。
- **不明は `[要記入: …]`**（価格・実績・社名）。埋めた風にしない。
- `knowledge/sales-domain/` があれば ICP・価格・差別化・反論処理に反映。無ければ該当を `[要記入]`。
