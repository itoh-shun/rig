# output-contract: spd-verdict

## output-contract: spd-verdict

SPDステークホルダー・レビューの出力構造。2段で定義する。**① 各視点レビュアーの出力**（機械抽出前提・**evidence-first**＝根拠を先に・判定は末尾）と、**② 親が集約するレポート**。前置き・挨拶・補足説明は禁止する。

---

## ① 視点レビュアーの出力（各 subagent が遵守）

6視点（hospital-executive / materials-manager / ward-nurse / spd-operator / spd-vendor-manager / distributor）の各レビュアーは、自分の1視点についてこの形式だけを返す。

```
観点: <hospital-executive|materials-manager|ward-nurse|spd-operator|spd-vendor-manager|distributor>

根拠: （1行。なぜその判定か）

良い点:
- （あれば。なければ「なし」）

懸念・要改善点:
- （直すべき点。誰が次に何をすべきかが分かる粒度で）

情報不足:
- （対象の記述の欠落で評価できなかった項目。なければ省略）

判定: <◎|○|△|×>
```

### ルール（視点レビュアー）

- **1行目は必ず `観点:`**。自分の担当視点名を出す。
- **判定は末尾に必ず出力する**（`判定:` で始まる行。最終行）。根拠を書いてから判定を出す。機械抽出は**最後に現れる** `判定:` 行を採用する。判定語は `◎`（この視点では申し分ない）/ `○`（合格）/ `△`（要改善）/ `×`（この視点にとって重大欠落）のいずれか。
- 根拠は **1行**。懸念・要改善点は実行可能な粒度（「現場負担に配慮を」ではなく「夜間の臨時請求手順を運用フローに追記する」）。
- 制度・事実（償還価格・薬機法・GS1 等）の誤りを見つけたら懸念に**事実誤認**として明示する。
- 評価できない項目を推測で埋めない。確認できなければ **情報不足**に明示する。
- **全体 150–300字**（日本語換算）。冗長な前置き・感想・締め挨拶は禁止。

---

## ② 親の集約レポート

親は6視点の verdict を集約し、次の形式で提示する。対象全文を親 context に抱えず、各 verdict の判定行・根拠・懸念点だけを読んで組み立てる。

```
## SPDステークホルダー・レビュー結果

総合判定: <GO|条件付きGO|要再検討>

| 視点 | 判定 | ひとことで |
|---|---|---|
| 病院経営層 | ◎○△× | （根拠1行） |
| 用度・材料部 | ◎○△× | （根拠1行） |
| 看護現場 | ◎○△× | （根拠1行） |
| SPD現場 | ◎○△× | （根拠1行） |
| SPD事業者経営 | ◎○△× | （根拠1行） |
| 卸・流通 | ◎○△× | （根拠1行） |

### 優先アクション（優先順）
1. （誰が・何を）
2. ...

### 情報不足（記述に足りず評価できなかった点）
- （あれば。追記を促す。なければ「なし」）
```

### 総合判定の決定ルール（親）

上位優先で最初に当てはまる段を採る。

- **GO**: `×` が無く、`△` が1つ以下。
- **条件付きGO**: `×` が無く、`△` が2つ以上（優先アクションの実施を条件として明示する）。
- **要再検討**: いずれかの視点に `×`（重大欠落）がある、または**事実誤認**（制度・償還・薬機法まわり）が指摘されている。

> 情報不足が過半で評価不能な視点が多い場合は、総合判定の代わりに「記述不足のため評価保留」と明示し、不足項目の追記を促す。
